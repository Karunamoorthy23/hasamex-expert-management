n8n Integration

Project → Expert Search → Email Invites

End-to-end n8n workflow for the Expert Management app

Executive Architecture Overview

System Overview

1. Trigger

Creating a Project in the app posts minimal project context to an n8n webhook.

2. Orchestration

n8n calls back the app to execute a “filter‑first” LinkedIn search using Apify and stores experts in the database.

3. Outreach

n8n (or the app) triggers invitation emails to lead experts with a link to the expert form.

Core Endpoints & Security

Authentication

n8n calls are protected with N8N_SERVICE_TOKEN. The app accepts it either as:

Header: Authorization: Bearer 
Query param: ?token=

See authorization logic in `search_experts` and `send_emails`.

Key Endpoints Map

  • Project creation: projects.py:create_project
  • n8n health/status: n8n_webhook.py:n8n_status
  • n8n → App (search): n8n_webhook.py:search_experts
  • n8n → App (emails): n8n_webhook.py:send_emails
  • BG invite sender: projects.py:_bg_send_project_invites

End‑to‑End Flow: Part 1

1) Create a New Project

  • Frontend posts to the app’s Project API, which persists the project and immediately notifies the n8n webhook in a background thread.
  • Payload includes: search‑relevant fields (title, target companies, geographies, functions, description).

Code path:
• Persistence: create_project
• Webhook notify: _notify_n8n_project_created

End‑to‑End Flow: Part 2

2) n8n Orchestrates Expert Search

n8n receives the webhook and calls /api/v1/n8n/search-experts. The app performs a structured search via Apify (LinkedIn):

  • Actor ID: configured as SEARCH_ACTOR
  • Structured payload: currentJobTitles, currentCompanies, locations, profileScraperMode: "Full"
  • Small batches (maxItems, takePages) keep queries targeted.

Full process detailed in apify_search_process.md

End‑to‑End Flow: Part 4

4) Prepare and Send Invite Emails

Execution Logic

  1. n8n calls /api/v1/n8n/send-emails
  2. App loads project, collects leads_expert_ids
  3. Converts leads → invited in UI
  4. Spawns background thread for transactional emails via Brevo/Resend.

Expert CTA & Forms

Emails include a link to the public Project Form:

/project-form/:project_id?expert_id=:expert_code
  • Fetch: get_project_form_public
  • Submit: submit_project_form_public

Simplified PPT Narrative

Step 1: Create Project in CMS

User fills required fields. App persists and notifies n8n webhook with search‑relevant payload.

Step 2: n8n Runs Filter‑Search

n8n triggers app. App queries Apify using filters, validates experience, scores relevance, stores Top 15.

Step 3: Lead Outreach

App updates lists (leads → invited) and emails experts. Experts click CTA and submit public form.

Step 4: Feedback Loop

Accepted/scheduled/completed calls update the project’s metrics in Engagement dashboards.

Ops: Config & Troubleshooting

Configuration Checklist

In `backend/flask.env`:

  • N8N_WEBHOOK_URL — incoming webhook
  • N8N_SERVICE_TOKEN — token for app endpoints
  • APIFY_API_TOKEN — LinkedIn actor access
  • BREVO_API_KEY — Email provider keys

Troubleshooting

  • 401 Unauthorized: Ensure N8N_SERVICE_TOKEN sent as Bearer or ?token=...
  • Empty/Mock Results: Check Apify token validity & actor limits; app provides high‑fidelity mock fallback.
  • Emails not sending: Verify EMAIL_PROVIDER keys; inspect logs in services/mailer.py.
