Low-Level Component Lifecycle

The sequence of operations when an expert profile is created or updated, highlighting document handling and data validation.

sequenceDiagram
    participant UI as React Form
    participant API as Flask Controller
    participant VAL as Logic & Validation
    participant FS as Expert PDF Folders

    API->>FS: Save file with UUID name

    API->>VAL: Clean Strings & Verify Email
    VAL->>DB: Check LinkedIn / Email Duplicates

Detailed Logic Workflow: Expert Onboarding

A step-by-step breakdown of the expert creation logic, including document processing and database persistence.

flowchart TD
    Start([User Clicks Save]) --> Valid{Frontend Validation}
    Valid -- Fail --> ShowErrors[Highlight Missing Fields]
    Valid -- Pass --> FileCheck{Document Selected?}
    
    FileCheck -- Yes --> Upload[POST /upload-pdf]
    Upload --> SaveFS[Store in /expert_pdf]
    SaveFS --> GetURL[Return Document URL]
    GetURL --> FinalPayload[Attach URL to Payload]
    
    FileCheck -- No --> FinalPayload
    
    FinalPayload --> PostExpert[POST /api/v1/experts]
    PostExpert --> DupCheck{Check Duplicates}
    
    DupCheck -- Found --> Conflict[409 Conflict Response]
    DupCheck -- Clear --> DBInsert[DB Insert Record]
    
    DBInsert --> Success[Redirect to Expert List]
    Conflict --> ShowWarning[Show Duplicate Warning Modal]

Technology Core

Frontend Stack

  • React 18: Modern component-driven hooks
  • Vite: Native ESM-based development server
  • Vanilla CSS: Performance-first custom styling

Backend Stack

  • Flask 3.0: Lightweight, scalable Python API
  • SQLAlchemy: Robust SQL toolkit & ORM
  • Pandas: High-speed Excel/CSV processing
